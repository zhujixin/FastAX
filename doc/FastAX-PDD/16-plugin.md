### 5.13 插件/扩展系统 (PRD §6.14 PLUG)

#### 5.13.1 中间件流水线

```
请求 → [Plugin A] → [Plugin B] → [核心路由] → [Plugin C] → 响应
           │              │                      │
       改写请求体       自定义认证             改写响应体
       添加 Header     限流策略              日志增强
```

| 插件类型 | 钩子点 | 用途 |
|---------|--------|------|
| beforeRequest | 路由决策前 | 请求改写、自定义认证、扩展限流 |
| afterRequest | 响应处理后 | 响应改写、日志增强、Webhook 通知 |
| route | 路由决策时 | 自定义路由算法 (AB 测试、自定义权重) |

#### 5.13.2 插件隔离

```
插件运行在沙箱环境:
  - 独立的 panic recover
  - 超时控制 (默认 500ms)
  - 资源限制 (CPU/内存)
  - 插件故障不影响核心转发链路
```
