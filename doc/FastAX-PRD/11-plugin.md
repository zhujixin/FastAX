### 6.14 插件/扩展系统（PLUG）

参考 Portkey Plugin Marketplace、LLMProxy 环状插件流水线设计。

| 功能 | 需求描述 | 优先级 | 备注 |
|------|----------|--------|------|
| PLUG-01 | **中间件流水线**：支持请求/响应中间件 Hook（beforeRequest / afterRequest），用户可自定义处理逻辑 | P2 | 可组合多个中间件 |
| PLUG-02 | **自定义路由策略插件**：支持插件化路由算法（优先级/成本优先/延迟优先/AB 测试），可通过脚本扩展 | P2 | — |
| PLUG-03 | **Webhook 转换**：请求/响应经过外部 Webhook 进行自定义转换，支持实时改写 | P2 | — |
| PLUG-04 | **限流策略插件**：可插拔的限流算法（令牌桶/漏桶/滑动窗口），支持按用户/Key/IP 多维度 | P2 | — |
| PLUG-05 | **监控插件接口**：对接 Prometheus、Datadog 等外部监控系统的插件接口 | P2 | — |
| PLUG-06 | **插件市场**：第三方开发者可提交插件，管理员在线安装/启用/禁用 | P3 | 长期规划 |
| PLUG-07 | **插件沙箱隔离**：插件运行在沙箱环境中，不直接影响核心转发链路稳定性 | P2 | — |
