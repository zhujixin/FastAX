## 13. 技术架构建议

### 12.1 推荐技术选型

| 层级 | 推荐方案 | 说明 |
|------|----------|------|
| **后端语言** | **Go 1.22+** | 全栈 Go，单体优先；代码按业务域划分 package，预留微服务拆分点 |
| **后端框架** | **Gin** | 高性能 Go HTTP 框架 |
| **数据库/ORM** | **SQLite 3.x (WAL 模式) + GORM** | 单 DB 文件，按 package 分表；预留拆分独立 DB 的接口层 |
| **缓存** | **Redis (go-redis)** | 用户会话、渠道缓存、配额缓存、限流计数 |
| **消息队列** | **RabbitMQ (amqp091-go)** | 异步解耦：通知、日志、风控、结算 |
| **搜索/日志** | **Elasticsearch** | 调用日志、审计日志聚合检索 |
| **前端框架** | **React 18 + TypeScript + i18next** | — |
| **前端 UI** | **Ant Design 5** | 企业级组件 |
| **前端构建** | **Vite** | — |
| **部署** | **单进程 + SQLite** | MVP 单体部署；模块通过 Go interface 解耦，后期可抽取为独立 gRPC 服务 |
| **部署** | **Docker + K8s** | 每服务独立容器化部署 |

### 12.3 翻译 Key 命名规范

```
<模块>:<子模块>.<路径>.<描述>
```

示例：
```json
{
  "token:product.buy_button": "立即购买",
  "token:product.price_label": "价格",
  "token:detail.expire_days": "还剩 {days} 天到期",
  "order:list.status.pending": "待支付",
  "order:list.status.paid": "已支付",
  "common:nav.login": "登录",
  "common:nav.register": "注册",
  "common:error.required": "此项为必填",
  "notification:email.reminder.subject": "Token 到期提醒"
}
```

### 12.4 语言回退链

```
zh-TW → zh-CN → en
zh-HK → zh-CN → en
en-US → en
en-GB → en
ja-JP → ja → en
ko-KR → ko → en
vi-VN → vi → en
th-TH → th → en
```

当浏览器语言为 `zh-TW` 时：
1. 查找 `zh-TW` 翻译 → 找到则使用
2. 未找到 → 回退到 `zh-CN` → 找到则使用
3. 仍未找到 → 回退到默认语言 `zh-CN`
4. 所有回退失败 → 显示 Key 名称（调试模式）或默认语言文案（生产模式）

## 14. UI/UX 设计指引

### 13.1 语言选择器

| 类型 | 位置 | 说明 |
|------|------|------|
| **Dropdown 选择器** | 页面顶部导航栏右侧 + 底部 Footer | 显示语言名称（母语称谓），如"中文"、"English"、"日本語" |
| **移动端** | 汉堡菜单 → 语言设置项 | — |
| **登录页** | 登录框下方 | 用户在登录前即需选择语言 |

### 13.2 设计注意事项

| 注意事项 | 说明 |
|----------|------|
| **文案长度变化** | 英文通常比中文长 30-50%（"提交"→"Submit"），按钮/卡片需预留弹性空间 |
| **日韩文本** | 日韩文字可能需要更大字号（部分汉字笔画密集），确保可读性 |
| **换行与截断** | 不使用固定宽度的单行截断（`text-overflow: ellipsis`），采用 `min-width` + 自适应 |
| **连接词处理** | 动态拼接文案时使用完整的模板而非字符串拼接 |
| **图标与文字** | 尽量使用通用图标降低文化差异，避免使用特定文化的手势/符号 |
| **时间表述** | 使用相对时间 + 绝对时间兜底，支持 Intl 格式化 |
