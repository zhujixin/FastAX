## 15. 实施路线图

### Phase 1：基础架构 + 中英双语（4-6 周）

| 阶段 | 任务 | 产出 |
|------|------|------|
| 第 1-2 周 | 转发引擎搭建（智能路由、故障转移、熔断器、健康检测）、i18n 框架选型 | 基础转发与 i18n 架构就绪 |
| 第 2-3 周 | 用户端核心页面（登录、商城、订单、个人中心）翻译 key 提取与中英翻译 | 用户端核心页面中英双语 |
| 第 3-4 周 | 管理后台核心页面翻译 key 提取与中英翻译 | 管理后台中英双语 |
| 第 4-5 周 | 通知模板（邮件、短信、站内信）中英双语 | 双语通知就绪 |
| 第 5-6 周 | 翻译文件构建与部署流程搭建 | 多语言构建流水线就绪 |

**Phase 1 完成标志**：用户可通过语言选择器切换中英文，全平台核心 UI 覆盖 ≥ 95%。

### Phase 2：Token 厂家入驻与销售平台（6 周）

| 阶段 | 任务 | 产出 |

| 第 3-4 周 | 自主定价、库存管理、供应商 API 注册与健康监控 | 供应商自主运营能力就绪 |

**Phase 2 完成标志**：Token 厂家可自助注册、上架商品、管理销售，平台自动完成结算。

### Phase 3：海外用户访问国内大模型（4 周）

| 阶段 | 任务 | 产出 |
|------|------|------|
| 第 1 周 | 国内节点部署、双轨智能路由搭建 | 海外→国内优化链路就绪 |
| 第 2 周 | 国内大模型 API 兼容适配（DeepSeek、Qwen、GLM 等转 OpenAI 格式） | 国内模型 API 兼容层上线 |

**Phase 3 完成标志**：海外用户可通过 FastAX 使用 Stripe 购买并使用国内大模型，端到端延迟 ≤ 1500ms。

| 阶段 | 任务 | 产出 |
|------|------|------|
