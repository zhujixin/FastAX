## 16. 附录：术语表

| 术语 | 定义 |
|------|------|
| **Token** | 数字资产或 API 调用的凭证单元，可以是 AI 模型调用额度、数字资产权益等 |
| **Token 代理** | 作为中转层，接收用户请求并转发至 Token 供应商，返回响应结果 |
| **Token 供应商** | Token 的源头提供方，如 OpenAI、Anthropic、Google 等 |
| **中转** | 用户请求经平台转发至最终供应商的过程 |
| **渠道** | 指具体的 Token 供应来源，一个供应商可能有多个渠道 |
| **渠道健康检测** | 定期检查各渠道的余额和可用性，确保服务稳定 |
| **流式输出** | 服务端逐步返回数据而非一次性返回全部结果，适用于实时场景 |
| **设备指纹** | 通过收集设备特征信息，生成唯一标识，用于识别恶意设备 |
| **风控引擎** | 基于规则和 AI 模型的风险检测系统 |
| **JWT** | JSON Web Token，用于身份认证的令牌 |
| **BCrypt** | 密码哈希算法，用于安全存储用户密码 |
| **AES-256** | 高级加密标准，256 位密钥长度的对称加密算法 |
| **OpenRouter 模式** | 参考 OpenRouter 的商业模式，按交易金额收取佣金 |
| **i18n** | Internationalization，国际化，使产品能适应不同语言和地区 |
| **L10n** | Localization，本地化，将产品翻译和适配到特定语言/地区 |
| **Locale** | 语言区域标识，如 `zh-CN`、`en-US`、`ja-JP` |
| **Translation Key** | 翻译条目的唯一标识符，如 `common:nav.login` |
| **源语言 (Source Language)** | 最初撰写文案的语言，本平台为 `zh-CN` |
| **目标语言 (Target Language)** | 翻译目标语言，如 `en`、`ja` |
| **回退语言 (Fallback Language)** | 当目标语言翻译缺失时使用的备用语言 |
| **RTL** | Right-to-Left，从右至左书写的语言（如阿拉伯语） |
| **XLIFF** | XML Localisation Interchange File Format，翻译行业标准交换格式 |
| **hreflang** | HTML 标签，告知搜索引擎页面面向的语言/地区 |
| **国内大模型** | 中国公司开发的大语言模型，如 DeepSeek、通义千问（Qwen）、智谱 GLM、文心一言等 |
| **双轨路由** | 同时支持海外→海外和海外→国内两种流向的智能链路选择机制 |
| **数据出境** | 个人信息和重要数据在中国境内收集后向境外提供的传输行为，需依法进行安全评估 |
| **生成式人工智能** | 中国《生成式人工智能服务管理暂行办法》监管范围内的 AI 内容生成服务 |
| **供应商入驻** | 第三方 Token 厂家在平台注册、通过审核后自主销售 Token 的机制 |
| **结算周期** | 平台与供应商之间进行交易结算的时间周期（如 T+7 或月结） |
| **平台佣金** | 平台从供应商交易额中抽取的服务费用，按百分比或固定费率计算 |
| **多协议原生** | API 网关直接支持各个供应商的原生协议（Anthropic Messages API、Gemini API），无需格式转换 |
| **模型后缀变体** | 通过 `model:variant` 语法指定路由策略，如 `:nitro`（最快）、`:floor`（最便宜）、`:thinking`（推理） |
| **BYOK** | Bring Your Own Key，用户使用自己的供应商 API Key 通过平台统一转发和监控 |
| **安全护栏（Guardrails）** | 对用户请求和模型响应进行安全检测的规则引擎，涵盖 PII、注入、内容审核等 |
| **Prompt 注入** | 恶意构造的输入试图让 AI 模型忽略原始指令或泄露敏感信息 |
| **PII** | Personally Identifiable Information，个人身份识别信息（邮箱、手机号、身份证等） |
| **语义缓存** | 基于语义相似度缓存 Prompt 响应，相同或近似请求直接返回缓存结果 |
| **Prompt Caching** | 利用上游供应商的缓存机制（针对重复 Prompt 前缀），减少输入 Token 消耗 |
| **SSO** | Single Sign-On，单点登录，企业用户通过 SAML/OIDC 一次登录访问所有资源 |
| **MCP** | Model Context Protocol，模型上下文协议，AI Agent 发现和调用外部工具的标准化协议 |
| **Rerank** | 重排序模型，对检索结果进行二次排序优化相关性 |
| **多模态** | 平台支持多种 AI 能力类型：文本、图片（生成/理解）、视频、音频（语音/音乐） |
| **模型市场** | 平台的模型发现和比较功能，展示价格、延迟、可用率等透明度数据 |
| **不可篡改审计日志** | 使用加密技术（AES-256-GCM + HMAC-SHA256）保护的日志，写入后不可修改或删除 |
